Refactoring Using Tests

Ever wondered how a test-driven developer works on PHP applications?

Have you ever heard this joke:

Q: What is legacy code?

A: Code that you wrote yesterday!

One of the bigger challenges a developer can face is taking a legacy application that has been up in production for a long time, working as required, and then have to extend it.

Without an easily-understood application structure and a test suite that can be run when required, you are going to have a hard time making all but the most trivial changes with any kind of confidence.

Chris created this video series to show you how to approach a legacy application that you understand how it works and refactor it to give it the structure and clarity so you can continue to refactor and add functionality with the knowledge your tests have your back when it comes to identifying regressions or misunderstandings of how your application works.

Recorded in DRM-free 1280 x 720 MPEG-4 with audio, you will watch Chris as he works and talks through his thought processes as he reworks a legacy PHP application to have a more modern structure and add effective unit tests to make future changes easier to do.
